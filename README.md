# Credit Card Fraud Detection

End-to-end anomaly detection system for financial transactions using an **Isolation Forest ensemble**. The primary objective is to **maximise Recall** — missing a fraudulent transaction is far costlier than a false alarm.

## Results

| Metric | Value |
|---|---|
| ROC-AUC | **0.9808** |
| Recall (fraud) | **1.000** |
| Precision (fraud) | 0.112 |
| F1 (fraud) | 0.201 |
| Average Precision | 0.463 |
| Fraud missed (FN) | **0 out of 151** |

## Approach

**Dataset:** 10,000 transactions · 8 original features → 18 after engineering · 1.51% fraud rate

**Model:** Ensemble of 5 Isolation Forest models (seeds 42, 123, 456, 789, 1024). Each model uses `n_estimators=500`, `max_features=0.8`, `bootstrap=True`. Anomaly scores are normalised per model and averaged.

**Threshold:** Selected at **Recall ≥ 0.99** on the precision-recall curve (~0.4443), rather than the default contamination cutoff.

**Engineered features (10):** `amount_log`, `trust_deficit`, `risk_score`, `amount_velocity_ratio`, `trust_velocity`, `is_night`, `combined_risk`, `high_velocity`, `low_trust`, `risk_x_amount`

## Quick Start

### 1. Clone & install

```bash
git clone https://github.com/YOUR_USERNAME/credit-card-fraud-detection.git
cd credit-card-fraud-detection
python -m venv venv
source venv/Scripts/activate        # Windows: .\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2. Get the data (DVC + Google Drive)

```bash
dvc remote modify --local myremote gdrive_client_id     YOUR_CLIENT_ID
dvc remote modify --local myremote gdrive_client_secret YOUR_CLIENT_SECRET
dvc pull
```

### 3. Train the ensemble

```bash
python -m src.models.train_ensemble
```

Saves `models/ensemble_isolation_forest.joblib` with all inference artifacts.

### 4. Run the API

```bash
uvicorn src.api.main:app --reload
```

Open **http://localhost:8000** for the transaction form UI, or use the Swagger docs at **http://localhost:8000/docs**.

## Docker

```bash
# Build (train the model first — step 3 above)
docker build -t fraud-detection .

# Run
docker run -p 8000:8000 fraud-detection
```

## API Reference

### `GET /`
Returns the HTML transaction form.

### `POST /predict`

**Request body:**

```json
{
  "amount": 5000.00,
  "transaction_hour": 2,
  "merchant_category": "Electronics",
  "foreign_transaction": 1,
  "location_mismatch": 1,
  "device_trust_score": 28,
  "velocity_last_24h": 7,
  "cardholder_age": 34
}
```

**Response:**

```json
{
  "is_fraud": 1,
  "anomaly_score": 0.812341,
  "threshold": 0.444258,
  "label": "FRAUD"
}
```

`anomaly_score` ranges 0–1; scores above `threshold` are flagged as fraud.

## Trade-offs & limitations

- The high-recall threshold (~0.4443) produces ~1,200 false positives per 10,000 transactions — these require manual review. This is a deliberate business decision.
- Isolation Forest is **unsupervised** and cannot use the fraud label during training. A supervised model (XGBoost, LightGBM) would achieve significantly higher precision at the same recall level.
- Monitor **score distribution drift** in production as transaction patterns evolve.
